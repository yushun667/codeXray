# CodeXray 操作手册

## 版本管理与 GitHub

### 远程仓库

- 地址：`git@github.com:yushun667/codeXray.git`
- 默认分支：`main`

### 推送到 GitHub

1. **首次推送前**：确保本机 SSH 已配置且 GitHub 公钥已添加。
   - 将 GitHub 加入已知主机（可选）：  
     `ssh-keyscan -t ed25519 github.com >> ~/.ssh/known_hosts`
   - 确认可连接：`ssh -T git@github.com`

2. **添加远程（若未添加）**：
   ```bash
   git remote add origin git@github.com:yushun667/codeXray.git
   ```

3. **提交并推送**（遵循项目规范：每次修改后 add + 中文 commit）：
   ```bash
   git add .
   git commit -m "描述本次修改内容"
   git push -u origin main
   ```

## CI 流程（GitHub Actions）

### 触发条件

- 向 `main` 或 `master` 分支 **push**
- 向 `main` 或 `master` 分支的 **Pull Request**

### 工作流文件

- 路径：`.github/workflows/ci.yml`
- 名称：`CI`

### 任务说明

| 任务 | 说明 |
|------|------|
| **扩展构建** | 在 Ubuntu 上使用 Node.js 20：`npm ci` → `npm run compile` → `npm run lint`（lint 失败不阻塞）。 |
| **解析器构建** | 在 Ubuntu 上安装 SQLite3、Protobuf、nlohmann-json、LLVM/Clang 14，在 `parser/` 下执行 CMake 配置与 Release 构建。 |

### 解析器 CI 依赖（Ubuntu）

- `build-essential`, `cmake`
- `libsqlite3-dev`
- `libprotobuf-dev`, `protobuf-compiler`
- `nlohmann-json3-dev`
- `clang-14`, `libclang-14-dev`, `llvm-14-dev`

### 查看结果

- 仓库 → **Actions** 页签：查看每次 push/PR 的 CI 运行记录与日志。
- 当前 CI 已配置为：**扩展构建**（Node 20 + npm ci + compile + lint）与 **解析器构建**（Ubuntu 22.04 + LLVM-14）均通过后，workflow 显示为成功。解析器构建失败时会自动输出 cmake/build 日志尾部便于排查。

## 打包与安装 VSCode 扩展（.vsix）

### 三平台 CI 打包（推荐）

GitHub 的 **Package** 工作流（`.github/workflows/package.yml`）在**推送到 main** 或**手动触发**时，会在三个 runner 上分别构建解析器并打包 .vsix，产物包含对应平台的解析器可执行文件：

| 产物名 | 平台 | Runner |
|--------|------|--------|
| codexray-linux-x64 | Linux x64 | ubuntu-22.04 |
| codexray-darwin-arm64 / codexray-darwin-x64 | macOS | macos-14 |
| codexray-win32-x64 | Windows x64 | windows-latest |

在仓库 **Actions → Package** 中选中一次运行，在摘要页底部 **Artifacts** 中下载对应平台的 .vsix，安装即可直接使用（无需单独构建解析器）。

若某平台构建失败，可下载该次运行的 **parser-build-logs-\*** 产物查看 cmake/build 日志；Linux 失败时还会在对应 job 的 **Summary** 中显示日志尾部。

### 本地打包（含解析器）

若需在本地打出**带解析器**的 .vsix（仅当前系统可用）：

1. 在 `parser/` 下完成解析器构建（见 01-解析引擎 文档）。
2. 在仓库根目录执行：
   ```bash
   mkdir -p bin
   cp parser/build/codexray-parser bin/   # Windows: cp parser/build/codexray-parser.exe bin/
   npm install
   npm run package
   ```
3. 得到 `codexray-0.1.0.vsix`（内含 `bin/codexray-parser`）。

若**不**复制解析器到 `bin/`，则 `npm run package` 仍会生成 .vsix，但不含解析器，需在扩展设置中指定解析器路径或本地构建后使用。

### 直接安装 .vsix

1. VSCode：`Ctrl/Cmd+Shift+P` → **“从 VSIX 安装扩展”** → 选择对应平台的 `.vsix`。
2. 命令行：`code --install-extension /path/to/codexray-0.1.0-<platform>.vsix`

### .vscodeignore 说明

- `parser/build/**`、`parser/third_party/**` 已排除，.vsix 中不包含构建中间产物和 LLVM 等第三方库。
- 仅将**当前平台**的解析器可执行文件放入 `bin/` 后打包，扩展通过 `getParserPath()` 优先使用 `bin/codexray-parser`（或 Windows 下 `bin/codexray-parser.exe`）。
