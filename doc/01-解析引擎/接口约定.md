# 解析引擎：接口约定

## 1. 调用方式

- 主仓库通过 **CLI** 或 **本地 IPC/HTTP** 调用解析引擎；推荐先实现 CLI，便于 Remote-SSH 与脚本化。

## 2. CLI 约定

### 2.1 解析

- **命令示例**：`codexray-parser parse --project <workspace_root> [--compile-commands <path>] [--output-db <db_path>] [--parallel N] [--lazy] [--priority-dirs <path1[,path2,...]>]`
- **参数**：
  - `--project`：工程根目录（必填）。
  - `--compile-commands`：compile_commands.json 路径；缺省时默认为 `<project>/compile_commands.json`。
  - `--output-db`：输出数据库路径；缺省可由主仓库约定（如 `<project>/.codexray/codexray.db`）。
  - **`--parallel`**：并行度，正整数。**默认**：`max(1, 系统 CPU 核心数 - 2)`。主仓库可传入配置值覆盖。
  - **`--lazy`**：启用**懒解析模式**。仅先解析「优先解析目录」对应的 TU；剩余目录在查询时才按需解析。缺省时可默认启用（或由主仓库配置决定）。
  - **`--priority-dirs`**：优先解析目录，多个路径用逗号分隔，相对工程根。仅当 `--lazy` 时生效；先只解析这些目录下的 TU，其余在查询时解析。
- **退出码**：0 成功，非 0 失败（建议在 doc 或 stderr 中约定错误码表）。
- **输出**：可选的 stdout 进度或摘要（如 JSON 行）；详细日志落盘由解析引擎或主仓库约定。
- **懒解析与查询**：启用懒解析时，查询接口（CLI 或 API）在发现目标符号/文件属于未解析范围时，应先触发对该范围（目录/TU）的解析、写 DB，再返回查询结果；主仓库调用查询时需允许解析引擎执行此类按需解析。

### 2.2 查询（若通过 CLI 暴露）

- 若查询也通过 CLI：`codexray-parser query --db <db_path> --project <workspace_root> --type call_graph|class_graph|data_flow|control_flow [--symbol <name>] [--file <path>] ... [--lazy] [--parallel N]`
- **懒解析**：传入 `--lazy` 或与 parse 阶段一致时，若查询命中的符号/文件属于尚未解析的目录，解析引擎应先对该范围执行按需解析（使用配置的优先解析目录与并行度），再返回查询结果。
- 返回：JSON 格式的图/表数据，便于主仓库在编辑区标签中渲染。
- 若改为进程内或本地 HTTP 查询 API，则与主仓库约定请求/响应 JSON schema，且 API 需支持「查询时触发按需解析」的语义（传入 project、priority-dirs、parallel 等以便补解析）。

## 3. 输出格式（解析摘要 / 查询结果）

- **解析摘要**（若有）：如 `{ "status": "ok", "files_parsed": N, "symbols": N, "errors": [] }`。
- **调用图查询**：节点列表 + 边列表，节点含 id、name、file、line 等，边含 caller、callee。
- **类图查询**：节点（类）+ 边（继承/组合/依赖），节点含 id、name、file 等。
- **数据流查询**：变量 + 读写点列表或边列表。
- **控制流查询**：基本块与边，块内语句或摘要。

（具体 JSON schema 可在实现时单独文件维护，并在此引用。）

## 4. 错误码建议

- 0：成功。
- 1：参数错误（如缺少 --project）。
- 2：找不到或无法读取 compile_commands.json。
- 3：Clang 解析失败（可带 stderr 信息）。
- 4：数据库写入失败。
- 5：查询失败（如 DB 不存在或查询参数无效）。

（实现时在 stderr 或日志中输出可读描述，便于主仓库展示给用户。）
