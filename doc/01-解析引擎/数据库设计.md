# 解析引擎：数据库设计

## 1. 选型与目标

- **选型**：建议 **SQLite**，单文件、无需独立服务，便于本地与 Remote-SSH 环境部署，主仓库或解析引擎均可直接读写。
- **目标**：持久化解析结果，支持按符号、文件、调用关系、类关系、数据流、控制流等高效查询（索引设计见下）。

## 2. 表结构（概要）

以下为逻辑表设计，具体字段与类型在实现时细化。

### 2.1 元数据与工程

- **project**：工程信息（id, root_path, compile_commands_path, parse_time, 等）。
- **file**：文件路径与工程关联（id, project_id, path, 等）。
- **parsed_file**（懒解析用）：记录已解析过的文件或目录范围（如 project_id, path 或 path_prefix, parsed_at），用于懒解析模式下判断某次查询是否需先对未解析范围执行按需解析；按需解析完成后更新此表。

### 2.2 符号与调用链

- **symbol**：函数/方法等符号（id, **usr** 统一符号解析唯一标识, name, qualified_name, kind, **def_file_id, def_line, def_column** 定义位置；可选 decl_file_id, decl_line, decl_column 声明位置；file_id/line/column 可为引用处或与 def 一致，以 def 为准用于跳转）。
- **call_edge**：调用边（caller_id, callee_id, call_site_file_id, call_site_line, call_site_column, **edge_type**：direct / via_function_pointer），用于构建调用图。直接调用一条边对应一个 callee；**经函数指针的间接调用**可为一条边对应一个“可能 callee”，即同一调用点可有多条 call_edge（caller 与 call_site 相同、callee 不同），以表示**函数指针所有可能的调用链**。
- **function_pointer_call**（可选，与 call_edge 二选一或互补）：若希望显式建模“一调用点 → 多可能目标”，可增加此表（call_site 标识, caller_id, possible_callee_ids 或多条 (caller_id, call_site, callee_id)），便于查询“该函数指针调用点所有可能走向”。
- 索引：symbol(**usr** 唯一), symbol(name, def_file_id, qualified_name)，call_edge(caller_id, callee_id)，call_edge(call_site_*, edge_type)。

### 2.3 类关系

- **class**：类/结构体（id, **usr**, file_id, name, qualified_name, **def_file_id, def_line, def_column** 定义位置，等）。
- **class_relation**：类间关系（parent_id, child_id, relation_type：inheritance / composition / dependency）。
- **class_member**：成员（class_id, symbol_id 或 type 等）。
- 索引：class(name, file_id)，class_relation(parent_id, child_id, relation_type)。

### 2.4 全局变量与数据流

- **global_var**：全局变量（id, **usr**, **def_file_id, def_line, def_column** 定义位置, file_id, name, 等）。
- **data_flow_edge**：数据流边（var_id, reader_id/writer_id, 读写位置等）。
- 索引：global_var(name, file_id)，data_flow_edge(var_id, reader_id, writer_id)。

### 2.5 控制流

- **cfg_node**：控制流节点/基本块（id, symbol_id, block_id, 等）。
- **cfg_edge**：控制流边（from_node_id, to_node_id, edge_type 等）。
- 索引：cfg_node(symbol_id)，cfg_edge(from_node_id, to_node_id)。

## 3. 查询场景示例

- 按 USR 或符号名查调用者/被调用者：查 symbol（含 usr、定义位置）+ call_edge；函数指针调用会得到多条边（同一 call_site 对应多个 possible callee）。
- 查某函数指针调用点的所有可能调用链：查 call_edge where edge_type = via_function_pointer 且 call_site 匹配，或查 function_pointer_call 表。
- 按类名查继承/组合关系：查 class + class_relation。
- 按全局变量查读写链：查 global_var + data_flow_edge。
- 按函数查 CFG：查 cfg_node + cfg_edge where symbol_id = ?。

## 4. 与主仓库的约定

- 数据库文件路径由主仓库或配置传入（如工作区 `.codexray/codexray.db`）；解析引擎负责创建/迁移 schema 与写入，主仓库或解析引擎提供只读查询 API 供扩展使用。
- Remote-SSH 下 DB 位于远程工作区路径，查询同样在远程执行。
