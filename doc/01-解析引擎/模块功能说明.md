# 解析引擎：模块功能说明

## 1. 模块定位

- **子仓库**：C/C++ 代码解析引擎（如 `codexray-parser`）。
- **技术**：C++，基于 **Clang libtooling**，依赖 `compile_commands.json` 驱动对工程的语义级分析。
- **输出**：函数调用链、类关系、全局变量数据流、控制流等结构化结果，并持久化到本地数据库，支持高效查询。

## 2. 功能列表

| 需求编号 | 功能 | 说明 |
|----------|------|------|
| 1.1 | 基于 compile_commands.json 解析 | 读取并解析 `compile_commands.json`，驱动 Clang 对工程进行完整/增量语义分析 |
| 1.1 | 函数调用链 | 提取并输出函数调用图（call graph），支持跨翻译单元（TU），可区分直接/间接调用 |
| 1.1 | 类关系 | 提取继承、组合、依赖等类关系，可输出为图或结构化数据（供可视化在编辑区标签中展示） |
| 1.1 | 全局变量数据流 | 分析全局变量的读写、传递关系，形成数据流图/表 |
| 1.1 | 控制流 | 函数内控制流图（CFG）：分支、循环、条件等基本块与边 |
| 1.2 | 解析结果持久化 | 将上述结果写入本地数据库（schema 见《数据库设计》），由本模块或与主仓库约定 |
| 1.3 | 高效查询 | 数据库选型与索引设计（如 SQLite + 索引），支持按符号、文件、调用关系、类名等查询 |

## 3. 实现要点

- **Clang libtooling**：使用 FrontendAction + AST 访问器（或 RecursiveASTVisitor）遍历 AST，收集函数声明/定义、调用表达式、类定义、基类、成员、全局变量引用等；需要时结合 CFG 接口生成控制流。
- **多 TU**：按 compile_commands.json 逐 TU 解析，跨 TU 的调用链与类关系在合并阶段通过符号名/路径解析关联。
- **数据库**：解析结果写入本地 DB（见 `doc/01-解析引擎/数据库设计.md`），通过 CLI 或查询 API 对外提供只读查询能力。
- **远程**：需提供可在 Linux 等远程环境运行的二进制（或源码在远程编译），供主仓库在 Remote-SSH 下调用。

## 4. 与主仓库的边界

- 主仓库负责：触发解析（传入工程根路径、compile_commands 路径、DB 路径）、轮询或回调解析进度、发起查询请求。
- 解析引擎负责：执行解析、写 DB、提供查询接口；不负责 UI 与 Agent 逻辑。
