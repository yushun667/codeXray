# 主仓库：命令与视图清单

以下为扩展在 `package.json` 的 `contributes` 中需要声明的命令、视图、设置、菜单等，便于实现与维护时对照。

## 1. 视图（Views）

| 视图 ID | 位置 | 说明 |
|---------|------|------|
| CodeXray 侧边栏 | 侧边栏 (Side Bar) | 与资源管理器、搜索等并列；内部为 **两个独立标签页**，通过标签切换 |
| CodeXray · 解析管理 | 侧边栏 · 标签页 1 | **「解析管理」** 标签页：工程选择、compile_commands 配置、解析触发、历史解析记录、查询树或搜索入口 |
| CodeXray · AI 对话 | 侧边栏 · 标签页 2 | **「AI 对话」** 标签页：输入框、发送、历史消息列表、引用当前符号/图 |

说明：侧边栏内 **AI 对话** 与 **解析管理** 为两个独立标签页，用户点击标签切换；可视化展示（调用链图、类图、数据流图、控制流图）在**编辑区新建标签**中打开（Custom Editor 或 Webview），需通过 `customEditors` 或 `commands` 打开对应标签。

**按钮与图标**：扩展内**所有按钮均使用 VSCode 图标**（Codicon / `vscode.ThemeIcon`）作为按钮图标，与编辑器风格一致，不使用自定义图片或 emoji。

## 2. 命令（Commands）

| 命令 ID | 说明 | 常见触发方式 |
|---------|------|----------------|
| codexray.openProject | 打开/选择工程目录 | 命令面板、侧边栏 |
| codexray.setCompileCommands | 设置 compile_commands.json 路径 | 设置或弹窗 |
| codexray.runParse | 执行解析（全量或增量，由配置 codexray.parserIncremental 或命令参数决定） | 命令面板、侧边栏按钮 |
| codexray.listParseHistory | 查看**历史解析记录**列表（调用解析引擎 list-runs 或查 DB 中 parse_run）；在侧边栏或面板展示 | 命令面板、侧边栏 |
| codexray.queryCallGraph | 查询并打开调用链图 | **代码编辑区（主）**：编辑器右键/命令，以当前符号为查询目标；命令面板、侧边栏为辅 |
| codexray.queryClassGraph | 查询并打开类关系图 | **代码编辑区（主）**：同上 |
| codexray.queryDataFlow | 查询并打开全局变量数据流 | **代码编辑区（主）**：同上 |
| codexray.queryControlFlow | 查询并打开控制流图 | **代码编辑区（主）**：同上 |
| codexray.openAIChat | 打开/聚焦 AI 对话视图 | 命令面板、活动栏 |
| codexray.focusVisualization | 聚焦到可视化标签 | 命令面板 |
| codexray.gotoSymbolInEditor | 从可视化定位到代码 | 由可视化标签内节点点击触发：主仓库接收 file/line/column，执行 `showTextDocument` 并定位到行/列，实现**可视化与代码互相交互** |

（具体 ID 以实际实现为准，此处为建议命名。查询类命令需支持 `when` 在编辑器焦点且为 C/C++ 文件时在编辑器右键菜单中展示。）

## 3. 设置（Configuration）

**解析引擎可执行体**：**不需要配置路径**；解析引擎可执行体**默认随扩展打包**，主仓库从扩展安装目录内加载使用，**用户无需单独安装**解析引擎。

| 配置键 | 类型 | 说明 |
|--------|------|------|
| codexray.compileCommandsPath | string | compile_commands.json 路径（可相对工作区根） |
| codexray.databasePath | string | 本地数据库路径（可选，默认在工作区 .codexray 下） |
| codexray.parserParallelism | number | 解析并行度，正整数；**默认**：`max(1, 系统核心数 - 2)`；传给解析引擎 `--parallel` |
| codexray.parserLazy | boolean | 是否启用**懒解析模式**；默认 `true`；启用时仅先解析优先目录，剩余目录在查询时才解析 |
| codexray.parserPriorityDirs | array of string | **优先解析目录**（相对工程根），如 `["src", "include"]`；懒解析下只先解析这些目录，其余在查询时按需解析 |
| codexray.parserIncremental | boolean | 是否默认使用**增量更新**解析；默认 `true`；为 true 时 runParse 传 `--incremental` 给解析引擎，仅解析变更文件并合并回 DB；支持**历史解析记录**与**增量更新**为必须能力 |
| codexray.agentEndpoint | string | Agent 服务地址（如 stdio 或 socket 路径 / HTTP URL） |
| codexray.llmProvider | string | **大模型 provider**（如 `openai`、`azure`、`ollama`）；必须支持设置大模型，与下列项共同生效 |
| codexray.llmModel | string | 大模型标识（如 `gpt-4`、`claude-3`、`qwen`、ollama 模型名） |
| codexray.llmEndpoint | string | 大模型 API 端点（OpenAI 兼容 URL 或 Ollama 等本地地址） |
| codexray.llmApiKey | string | API Key（可选；敏感项建议用 SecretStorage 或环境变量，不强制明文配置） |
| codexray.logLevel | string | 日志级别 |
| codexray.logPath | string | 日志落盘路径（可选） |

## 4. 菜单（Menus）

- **命令面板**：上述命令均可通过 `commandPalette` 暴露。
- **侧边栏**：视图的上下文菜单（若有刷新、重新解析等）。
- **代码编辑区（查询主入口）**：在 C/C++ 编辑器内，对当前光标/选中符号的**右键上下文菜单**中必须提供「查看调用链」「查看类关系」「查看数据流」「查看控制流」等入口（与上述 query 命令绑定），使查询入口在代码编辑区；可选「在 AI 对话中引用当前符号」。`when` 条件建议为编辑器焦点且语言为 c/cpp。
- **资源管理器**：可选从文件节点右键发起解析或打开可视化。
- **可视化与代码交互**：可视化标签内节点点击时，通过扩展 API 调用 `codexray.gotoSymbolInEditor` 或等价逻辑，由主仓库打开对应文件并定位到行/列。

## 5. 状态栏（Status Bar）

- **解析进度**：扩展在状态栏注册一项（如 `codexray.parseProgress`），**必须显示解析进度百分比**（例如「CodeXray 解析 45%」）；解析进行中定期从解析引擎或主仓库轮询/回调获取进度并更新；解析结束或失败后更新为「完成」或错误态。实现时使用 VSCode Status Bar API（`window.createStatusBarItem`）展示百分比文案。

## 6. Remote 能力

- 在 `package.json` 中声明 **extensionKind**，支持在 Remote-SSH 工作区运行（如 `"extensionKind": ["ui", "workspace"]` 或 `["workspace"]`）。
- 上述视图、命令、设置、状态栏在本地与远程环境下均可用，行为一致。
