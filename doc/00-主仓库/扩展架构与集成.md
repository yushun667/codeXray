# 主仓库：扩展架构与集成

## 1. 主仓库定位

- **主仓库**即 VSCode 平台扩展代码仓（如 `CodeXray` / `codexray-extension`）。
- 职责：扩展清单、激活逻辑、命令/菜单、与三个子组件的集成（调用解析引擎、加载 UI、与 Agent 通信）。
- 通过子模块或 npm/cargo 依赖引用三个子仓库，扩展运行时由主仓库协调解析引擎、UI 与 Agent 服务。

## 2. 仓库与依赖关系

```
主仓库 (VSCode 扩展)
├── 引用 子仓库1：C/C++ 解析引擎（CLI 或本地 API）
├── 引用 子仓库2：可视化界面（npm 包或子模块，侧边栏 + 编辑区标签）
└── 引用 子仓库3：AI Agent 管理模块（本地服务：stdio/socket/HTTP）
```

- **解析引擎**：主仓库通过 CLI 或本地 IPC/HTTP 调用，传入工程路径与 `compile_commands.json`，以及**并行度**（默认核心数-2）、**是否懒解析**、**优先解析目录**（见 `doc/01-解析引擎/接口约定.md`），获取解析结果或查询 DB；懒解析下查询可能触发对剩余目录的按需解析。
- **可视化界面**：主仓库在**侧边栏**加载 AI 对话 UI，在**编辑区新标签**中加载可视化视图（调用链/类图/数据流/控制流），通过 VSCode 消息与数据接口与扩展后端通信。
- **Agent 模块**：主仓库与侧边栏 AI 对话 UI 通过当前环境的 Agent 服务（stdio 或 socket/HTTP）通信，发送消息、接收回复、可选写入知识库。

## 3. 数据流与调用关系

- 用户选择工程 / 触发解析 → 主仓库调用解析引擎（本地或远程）→ 解析结果写入当前环境 DB → 通知 UI 刷新。
- 用户发起查询（调用链/类关系/数据流/控制流）：**查询主入口在代码编辑区**（对当前符号右键或命令），主仓库根据编辑器内符号解析查询条件，从当前环境 DB 查询 → 在编辑区新标签中渲染图/树/列表。**必须支持可视化与代码互相交互**：从可视化节点点击可定位到代码（主仓库调用 `showTextDocument` 并定位到行/列）；从代码到可视化由编辑器右键/命令打开图（详见 `doc/02-可视化界面/UI与交互逻辑.md`）。
- 用户在侧边栏 AI 对话输入 → 主仓库将消息转发给当前环境 Agent 服务（带上下文如当前文件/符号）→ Agent 使用记忆与本地知识库生成回复 → 流式返回 → 侧边栏内展示。

## 4. 激活与生命周期

- 扩展激活时：使用**扩展内打包的解析引擎可执行体**（无需用户配置路径或单独安装）、启动或连接 Agent 服务、打开 DB 或通过解析引擎查询接口就绪。
- 在 **Remote-SSH** 场景下：上述组件均在**远程主机**上运行，主仓库在远程扩展主机上激活，UI 通过 VSCode 远程通道与后端通信。

## 5. Remote-SSH 与运行环境

- 扩展**必须支持通过 Remote-SSH 连接远程服务器**，且在远程环境中的功能与本地一致。
- **运行模型**：用户通过 Remote-SSH 打开远程工作区时，扩展在**远程扩展主机**上激活；解析引擎、本地数据库、Agent 服务均在**远程主机**上运行；侧边栏与编辑区标签的 UI 仍由 VSCode 前端渲染，通过远程通道与后端通信。
- **一致性**：远程服务器上的操作（选择工程、解析、查询、打开可视化标签、侧边栏 AI 对话、记忆与知识库读写）与本地行为相同，无需为远程单独实现一套逻辑。
- **实现要点**：
  - `package.json` 中声明 `extensionKind`（如 `["ui", "workspace"]` 或 `["workspace"]`），使扩展在远程工作区侧运行。
  - 解析引擎可执行体随扩展打包（无需单独安装），主仓库按当前环境（本地或远程）从扩展目录加载对应平台二进制；Agent 需提供可在 Linux 等远程环境运行的二进制（或源码在远程编译）。数据库路径、Agent 通信地址等基于当前工作区根路径或用户配置，自动适配本地/远程。
- **部署要求**：文档中需说明 Remote-SSH 下远程主机需具备的条件（如安装 Clang、Rust 运行时等）以及网络/权限约束；各 UI 在本地与远程下的位置与行为一致（见 `doc/02-可视化界面/UI与交互逻辑.md`）。

## 6. 日志与错误

- 统一日志落盘，关键步骤与错误可追溯（满足高效日志系统规则）。
- **解析进度**在**状态栏**中展示，**必须显示百分比**（如「CodeXray 解析 45%」）；解析错误、Agent 连接状态、数据库路径等在设置页或状态栏展示；本地/远程状态需可区分显示。
