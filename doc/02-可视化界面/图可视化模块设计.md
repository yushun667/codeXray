# 图可视化模块设计文档（AntV G6 v5 实现）

## 1. 模块概述

本模块为 CodeXray VSCode 扩展的函数调用图可视化功能，使用 **AntV G6 v5** 作为图渲染引擎，在 VSCode Webview 中展示函数调用关系的交互式图形界面。

### 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| AntV G6 | ^5 | 图渲染引擎（Canvas/WebGL） |
| @dagrejs/dagre | ^1.1 | 层次布局算法（注入 rank 约束） |
| React 18 | ^18.2 | UI 组件框架 |
| TypeScript | ^5.0 | 类型安全 |
| Vite 5 | ^5.0 | 构建工具 |

## 2. 文件结构

```
resources/ui/src/visualization/
├── graphEntry.tsx          # 入口（createRoot → GraphPage）
├── GraphPage.tsx           # 图页面主组件（消息处理、状态管理）
├── G6Graph.tsx             # G6 核心组件（Graph 实例生命周期、事件、工具栏）
├── GraphContextMenu.tsx    # 右键菜单（单节点 + 框选批量）
├── g6Config.ts             # G6 配置工厂（节点/边/行为/插件）
├── g6Adapter.ts            # 数据格式转换 + 增量合并
├── g6Layout.ts             # 自定义双向 dagre 布局（BFS rank 约束）
├── g6.css                  # 画布 + 工具栏样式（VSCode 主题变量）
└── adapters/
    ├── types.ts            # 中性节点/边类型（图库无关）
    ├── callGraph.ts        # call_graph → AdapterNode/AdapterEdge
    ├── classGraph.ts       # class_graph → AdapterNode/AdapterEdge
    ├── dataFlow.ts         # data_flow → AdapterNode/AdapterEdge
    └── controlFlow.ts      # control_flow → AdapterNode/AdapterEdge
```

## 3. 架构设计

### 3.1 数据流

```
host (initGraph/graphAppend)
  ↓ postMessage
GraphPage.tsx
  ↓ adaptGraph()
AdapterNode<FlowNodeData>[] + AdapterEdge[]
  ↓ toG6Data()
G6 NodeData[] + EdgeData[]
  ↓ g6Ref.setData() / appendData()
G6Graph.tsx
  ↓ graph.setData() + graph.render()
AntV G6 Graph 实例
  ↓ bidirectional-dagre 布局
Canvas 渲染
```

### 3.2 布局算法

**双向树状布局**（`bidirectional-dagre`）：

1. **BFS 计算有向层级**：从根节点集合出发
   - 正向 BFS（沿 source→target）：callees 层级 +1, +2, ...
   - 反向 BFS（沿 target→source）：callers 层级 -1, -2, ...
   - 根节点自身层级为 0

2. **注入 dagre rank 约束**：将 BFS 层级作为 dagre 的 `rank` 参数
   - dagre rank 必须 ≥ 0，故加偏移量 `rankOffset = -minRank`
   - 方向 LR（左到右），层间距 120px，同层间距 40px

3. **根节点居中**：根在 rank=0（偏移后为中间列），callers 在左，callees 在右

### 3.3 节点配置

| 属性 | 值 |
|------|-----|
| 类型 | rect |
| 宽度 | 280px |
| 高度 | 动态（≥60px，按 label 行数计算） |
| 圆角 | 8px |
| 根节点色 | rgb(193, 125, 55) 橙色 |
| 普通节点色 | rgb(81, 154, 186) 蓝色 |
| 端口 | 4 个（上右下左），边自动选择最近端口 |

### 3.4 交互功能

| 功能 | 实现方式 |
|------|----------|
| 拖拽画布 | G6 内置 `drag-canvas` 行为 |
| 滚轮缩放 | G6 内置 `zoom-canvas` 行为，范围 5%~200% |
| 点击选中 | G6 内置 `click-select` 行为 |
| Shift 框选 | G6 内置 `brush-select` 行为 |
| 节点拖拽 | G6 内置 `drag-element` 行为 |
| 路径高亮 | BFS 最短路径 + `setElementState` highlight/dimmed |
| 撤销/恢复 | G6 History 插件，Ctrl+Z / Ctrl+Shift+Z |
| Delete 删除 | 键盘监听，删除选中节点及关联边 |
| 双击跳转 | `node:dblclick` → postMessage `gotoSymbol` |
| 右键菜单 | `node:contextmenu` → React 覆盖层 |
| 适应画布 | 工具栏按钮 → `graph.fitView()` |
| 重新布局 | 工具栏按钮 → `graph.layout()` + `graph.fitView()` |

### 3.5 右键菜单

**单节点菜单**（GraphContextMenu）：
- 查询调用节点 → `queryPredecessors`
- 查询被调用节点 → `querySuccessors`
- 跳转到定义 → `gotoSymbol`
- 删除节点 → `removeNode`

**框选菜单**（SelectionContextMenu）：
- 删除选中的 N 个节点 → `removeNodes`

## 4. 与主仓库协议

协议不变，通过 `postMessage` 通信：

```
host → webview:
  initGraph   { graphType, querySymbol, queryFile, nodes, edges }
  graphAppend { nodes, edges }

webview → host:
  graphReady
  gotoSymbol        { file, line, column }
  queryPredecessors  { graphType, nodeId, symbol, filePath }
  querySuccessors    { graphType, nodeId, symbol, filePath }
```
