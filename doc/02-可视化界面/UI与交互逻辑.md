# 可视化界面：UI 与交互逻辑

本文档明确所有交互 UI 的**位置**、**触发条件**、**数据来源与去向**、**与三个子仓库的调用关系**。本地与远程（Remote-SSH）下位置与行为一致。

---

## 1. 布局约定

| UI | 位置 | 说明 |
|----|------|------|
| **侧边栏** | 与资源管理器、搜索等并列，通过活动栏图标打开 | 侧边栏内为 **两个独立标签页**：**「解析管理」** 与 **「AI 对话」**，用户通过标签切换 |
| **解析管理** 标签页 | 侧边栏 · 第 1 个标签 | 工程选择、compile_commands 配置、解析触发、历史解析记录、辅助查询入口（树或搜索框）等 |
| **AI 对话** 标签页 | 侧边栏 · 第 2 个标签 | AI 对话窗口：输入框、发送、历史消息列表、引用当前符号/图 |
| 可视化展示（调用链/类图/数据流/控制流） | **编辑区**，**新建标签页** | Custom Editor 或 Webview 作为编辑器 Tab，与代码文件标签并列，可多标签切换 |
| **查询主入口** | **代码编辑区** | 在编辑器内对符号（函数/类/变量等）右键或命令触发查询，在编辑区新标签打开可视化；**必须支持可视化与代码互相交互**（见第 3、3.1 节） |
| 解析进度（**百分比**）、错误、Agent 状态、DB 路径等 | **状态栏**（解析进度必须在此显示**百分比**）、设置页 | 本地/远程状态可区分显示 |

### 1.1 按钮与图标规范

- **所有按钮均使用 VSCode 图标作为按钮图标**，与编辑器风格一致。
- 实现时使用 VSCode 扩展 API 提供的 **Codicon**（如 `vscode.Codicon`）或 **ThemeIcon**（`vscode.ThemeIcon`），不使用自定义图片、外部图标库或 emoji 作为按钮图标。
- 适用于：侧边栏「解析管理」「AI 对话」标签页内的按钮（如解析、发送、刷新、引用当前符号等）、编辑区可视化标签内的操作按钮、以及扩展内所有其它按钮。

## 2. 工程选择与解析触发

| 项 | 说明 |
|----|------|
| **界面位置** | 侧边栏 **「解析管理」标签页**、命令面板、设置页（compile_commands 路径） |
| **触发条件** | 用户选择工程根目录或填写 compile_commands.json 路径后，点击「解析」或执行对应命令 |
| **数据来源与去向** | 用户输入 → 主仓库 → 调用解析引擎（子仓库 1）CLI/API，传入工程路径与 DB 路径；解析结果写入当前环境 DB；主仓库收到进度/完成/失败后更新 UI |
| **与子仓库关系** | 主仓库调用解析引擎；UI 仅与主仓库通信，不直接访问解析引擎 |

## 3. 调用链 / 类关系 / 数据流 / 控制流 查询与展示

| 项 | 说明 |
|----|------|
| **查询入口（主）** | **必须在代码编辑区**：用户在编辑器内对当前光标/选中的符号（函数、类、全局变量等）通过**右键菜单**或**命令**发起查询（如「查看调用链」「查看类关系」「查看数据流」「查看控制流」）；主仓库根据当前符号解析查询条件，发起查询并在**编辑区新建标签**中打开对应可视化。侧边栏的树或搜索框作为辅助查询入口。 |
| **界面位置** | 图或列表在**编辑区新建标签**中展示（每个查询可打开一个或多个标签，如「调用链图」「类关系图」等） |
| **触发条件** | 用户在**编辑器内**对符号右键或执行命令；或在侧边栏 **「解析管理」标签页** 输入/选择查询条件；主仓库发起查询并在编辑区新标签打开结果 |
| **数据来源与去向** | 查询条件（含从编辑器传入的符号位置/名称）→ 主仓库 → 从当前环境 DB 查询（或调用解析引擎查询 API）→ 返回 JSON → 主仓库将数据传给编辑区标签中的可视化组件 → 渲染图/树/列表；支持从节点下钻（如点击函数再查调用者/被调用者） |
| **与子仓库关系** | 数据来自解析引擎持久化的 DB 或解析引擎查询接口；可视化组件（子仓库 2）只负责渲染，由主仓库拉取数据并注入 |
| **四种图与 Adapter** | 四种图（call_graph / class_graph / control_flow / data_flow）的 Parser 边字段、前端 dataAdapter 转换规则、图内中文标题映射见 [主仓库 UI 元素记录](../00-主仓库/UI元素记录.md) 第 2.1 节 |
| **默认深度与扩展** | **默认仅查询并展示 3 层图**；图中节点支持**折叠/展开**；**右键节点**可「继续查询前置节点」「继续查询后置节点」，主仓库返回增量数据后**在当前图界面合并**（不新开标签），详见 [可视化界面详细功能与架构设计](可视化界面详细功能与架构设计.md) |

### 3.1 可视化与代码互相交互（必须支持）

| 方向 | 交互方式 | 说明 |
|------|----------|------|
| **代码 → 可视化** | 在编辑区对符号右键或执行命令 | 以当前光标/选中符号为查询目标，在编辑区新标签打开调用链图/类图/数据流图/控制流图；若该符号无对应类型（如对变量查控制流），可提示或忽略 |
| **可视化 → 代码** | 点击可视化中的节点（函数、类、变量、基本块等） | 主仓库接收节点对应的 file / line / column（或 symbol id 查出的位置），调用 VSCode API **打开对应文件并定位到行（及列）**，实现从图跳转到代码；若节点对应多位置（如重载），可列表选择或打开第一处 |
| **双向联动** | 编辑器与可视化标签同时打开时 | 可选：在编辑器中切换符号时，若已有对应可视化标签，可高亮或聚焦图中对应节点；在图中点击节点定位到代码时，编辑器获得焦点并选中对应符号，形成闭环 |

以上交互在本地与 Remote-SSH 下行为一致；主仓库负责解析编辑器中的符号、将节点位置信息与 VSCode 的 `vscode.window.showTextDocument` 等 API 对接。

## 4. AI 对话窗口（侧边栏 · 「AI 对话」标签页）

| 项 | 说明 |
|----|------|
| **界面位置** | 侧边栏 **「AI 对话」标签页** 内的 Webview 或面板：输入框、发送按钮、历史消息列表、可选「引用当前符号/当前图」按钮 |
| **触发条件** | 用户输入问题并发送；可选点击「引用当前符号」将当前编辑器符号或当前打开的可视化图作为上下文附加 |
| **数据来源与去向** | 用户输入 + 可选上下文 → 主仓库 → 转发给当前环境的 Agent 服务（子仓库 3），带当前文件/符号等信息 → Agent 使用记忆与本地知识库生成回复 → 流式或一次性返回 → 主仓库将回复推送到侧边栏 UI 展示；可选「将当前回答或解析结果写入知识库」由主仓库请求 Agent 完成 |
| **与子仓库关系** | 主仓库与 Agent 服务（子仓库 3）通信；侧边栏「AI 对话」标签页内 UI（子仓库 2）只负责展示与输入，不直接连 Agent |
| **消息展示格式** | 用户与 Agent 的消息内容均以 **Markdown** 格式渲染（代码块、列表、加粗、链接等），实现见 [可视化界面详细功能与架构设计](可视化界面详细功能与架构设计.md) 2.6 节 |

## 5. 设置与状态

| 项 | 说明 |
|----|------|
| **界面位置** | **解析进度**：在**状态栏**中展示，**必须显示百分比**（如「CodeXray 解析 45%」）；解析错误提示（通知或面板）、Agent 服务连接状态、数据库路径等，在设置页或状态栏展示；**本地/远程状态需可区分显示**（如标识「远程」或当前工作区类型） |
| **触发条件** | 解析进行中状态栏自动更新进度百分比；解析结束或失败后更新为完成/错误状态；Agent 未启动时提示并引导启动或自动启动 |
| **数据来源与去向** | 主仓库从解析引擎与 Agent 连接状态获取；用户可在设置中修改路径、端点等配置 |
| **与子仓库关系** | 主仓库汇总解析引擎与 Agent 的状态与配置，统一展示 |

---

## 6. 本地与远程一致性

- 以上 UI 在 **Remote-SSH** 下的**位置与行为**与本地一致：侧边栏仍为两个独立标签页（解析管理、AI 对话），编辑区标签仍在编辑区；仅数据来源变为远程主机上的 DB 与 Agent 服务，由主仓库在远程扩展主机上调用并经由 VSCode 远程通道与前端通信。
