# 可视化界面：模块功能说明

## 1. 模块定位

- **子仓库**：可视化界面（如 `codexray-ui`）。
- 可做成 npm 包或子模块，被主仓库引用；提供图与 AI 对话等视图的 UI 实现。
- **布局**：**侧边栏**内为 **两个独立标签页**——**「解析管理」** 与 **「AI 对话」**，用户通过标签切换；调用链/类图/数据流/控制流等可视化在**编辑区新建标签**中展示。

## 2. 功能列表

| 需求编号 | 功能 | 说明 |
|----------|------|------|
| 2.1 | 可视化查询结果 | 展示调用链、类图、数据流图、控制流图（树、图、列表等形式），在编辑区以新标签打开 |
| 2.1 | AI 对话窗口 | 与 Agent 模块对接的对话 UI（输入、发送、历史消息列表、流式输出），在侧边栏 **「AI 对话」标签页** 展示 |
| 2.1 | 查询入口在代码编辑区 | 查询主入口必须在代码编辑区：通过编辑器内对符号的右键菜单或命令触发查询，在编辑区新标签打开可视化 |
| 2.1 | 可视化与代码互相交互 | 必须支持：从代码到可视化（符号 → 打开图）；从可视化到代码（点击图节点 → 打开文件并定位到行/列）；可选双向联动（编辑器与图互相同步高亮/定位） |

## 3. 视图类型与数据来源

- **侧边栏**（两个独立标签页）：
  - **解析管理** 标签页：工程选择、解析触发、历史解析记录、辅助查询入口（树或搜索框）；数据来源为主仓库（工作区、解析进度、DB、解析历史）。
  - **AI 对话** 标签页：聊天界面；数据来源为 Agent 服务（历史、流式回复），主仓库负责转发消息与上下文（当前文件/符号）。
- **编辑区标签**：
  - **调用链图**：节点为函数，边为调用关系；数据来源为解析引擎 DB 的查询结果（主仓库或解析引擎查询 API）。节点可点击，跳转到代码对应位置。
  - **类关系图**：节点为类，边为继承/组合/依赖；数据来源同上；节点可点击定位到代码。
  - **数据流图**：全局变量与读写关系；数据来源同上；节点可点击定位到代码。
  - **控制流图**：函数内 CFG；数据来源同上；节点可点击定位到代码。
- **查询主入口**：代码编辑区（对符号右键或命令）触发查询；侧边栏「解析管理」标签页为辅助入口。

## 4. 与主仓库的对接方式

- 主仓库在**侧边栏**注册一个容器视图，内部分为 **「解析管理」** 与 **「AI 对话」** 两个独立标签页，分别挂载本模块提供的解析管理 UI 与 AI 对话 Webview；在**编辑区**通过 Custom Editor 或 Webview 作为 Tab 打开可视化内容。
- 若子仓库为独立 npm 包：提供 React/Vue 等组件或打包后的静态资源，主仓库负责加载与消息传递（postMessage / 数据绑定）。
- 与解析引擎/数据库：不直接访问 DB，通过主仓库或解析引擎提供的查询接口获取 JSON，再渲染图/树/列表。
- 与 Agent：不直接连 Agent，通过主仓库转发用户输入与接收回复（含流式），在侧边栏内展示。
- **UI 规范**：**所有按钮均使用 VSCode 图标**（Codicon / ThemeIcon）作为按钮图标，不使用自定义图片或 emoji，详见 `doc/02-可视化界面/UI与交互逻辑.md` 第 1.1 节。

## 5. 本地与远程一致性

- 在 Remote-SSH 下，侧边栏与编辑区标签的**位置与行为**与本地一致：侧边栏仍为两个独立标签页（解析管理、AI 对话），可视化仍在编辑区新标签中打开；数据来自当前环境（远程）的 DB 与 Agent 服务。
