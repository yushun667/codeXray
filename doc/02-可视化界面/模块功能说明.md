# 可视化界面：模块功能说明

## 1. 模块定位

- **子仓库**：可视化界面（如 `codexray-ui`）。
- 可做成 npm 包或子模块，被主仓库引用；提供图与 AI 对话等视图的 UI 实现。
- **布局**：AI 对话窗口在**侧边栏**；调用链/类图/数据流/控制流等可视化在**编辑区新建标签**中展示。

## 2. 功能列表

| 需求编号 | 功能 | 说明 |
|----------|------|------|
| 2.1 | 可视化查询结果 | 展示调用链、类图、数据流图、控制流图（树、图、列表等形式），在编辑区以新标签打开 |
| 2.1 | AI 对话窗口 | 与 Agent 模块对接的对话 UI（输入、发送、历史消息列表、流式输出），在侧边栏展示 |

## 3. 视图类型与数据来源

- **侧边栏**：
  - **工程/解析视图**：树或列表展示工程、解析状态；数据来源为主仓库（工作区、解析进度、DB 是否就绪）。
  - **AI 对话视图**：聊天界面；数据来源为 Agent 服务（历史、流式回复），主仓库负责转发消息与上下文（当前文件/符号）。
- **编辑区标签**：
  - **调用链图**：节点为函数，边为调用关系；数据来源为解析引擎 DB 的查询结果（主仓库或解析引擎查询 API）。
  - **类关系图**：节点为类，边为继承/组合/依赖；数据来源同上。
  - **数据流图**：全局变量与读写关系；数据来源同上。
  - **控制流图**：函数内 CFG；数据来源同上。

## 4. 与主仓库的对接方式

- 主仓库在**侧边栏**注册视图并挂载本模块提供的 Webview 或 Tree 数据源；在**编辑区**通过 Custom Editor 或 Webview 作为 Tab 打开可视化内容。
- 若子仓库为独立 npm 包：提供 React/Vue 等组件或打包后的静态资源，主仓库负责加载与消息传递（postMessage / 数据绑定）。
- 与解析引擎/数据库：不直接访问 DB，通过主仓库或解析引擎提供的查询接口获取 JSON，再渲染图/树/列表。
- 与 Agent：不直接连 Agent，通过主仓库转发用户输入与接收回复（含流式），在侧边栏内展示。

## 5. 本地与远程一致性

- 在 Remote-SSH 下，侧边栏与编辑区标签的**位置与行为**与本地一致：侧边栏仍为侧边栏，可视化仍在编辑区新标签中打开；数据来自当前环境（远程）的 DB 与 Agent 服务。
