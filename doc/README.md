# CodeXray 文档索引

本目录存放 CodeXray VSCode 扩展及相关子模块的设计与说明文档。

## 文档结构

| 目录/文件 | 说明 |
|-----------|------|
| [00-主仓库/](00-主仓库/) | 主仓库（VSCode 扩展）架构、集成方式、命令与视图清单、**详细功能与架构设计**（500-800 行/模块粒度） |
| [01-解析引擎/](01-解析引擎/) | C/C++ 解析引擎（Clang libtooling）功能、数据库设计、接口约定、**解析引擎详细功能与架构设计**（500-800 行/模块粒度） |
| [02-可视化界面/](02-可视化界面/) | 可视化界面模块功能、UI 元素与交互逻辑、**详细功能与架构设计**（500-800 行/模块粒度、postMessage 协议与图数据格式） |
| [03-Agent模块/](03-Agent模块/) | AI Agent 管理模块（Rust）功能、接口与协议、**记忆管理策略**（短期/长期保留与淘汰、容量、协同）、**Agent模块详细功能与架构设计**（500-800 行/模块粒度）、**知识库树状结构与节点设计**（树状摘要、多格式与 OCR、节点结构） |
| [04-系统设计/](04-系统设计/) | 系统框架总览、数据流、技术选型 |
| [占位功能记录.md](占位功能记录.md) | 当前仅占位、尚未完成实现的功能登记 |

## 阅读顺序建议

1. **系统总览**：先读 [04-系统设计/系统框架与数据流.md](04-系统设计/系统框架与数据流.md)。
2. **扩展集成**：再读 [00-主仓库/扩展架构与集成.md](00-主仓库/扩展架构与集成.md) 与 [00-主仓库/命令与视图清单.md](00-主仓库/命令与视图清单.md)。
3. **主仓库实现**：开发扩展时按 [00-主仓库/主仓库详细功能与架构设计.md](00-主仓库/主仓库详细功能与架构设计.md) 的模块与接口实现（每模块约 500-800 行）。
4. **各子模块**：按需查阅 01 / 02 / 03 下对应文档；实现解析引擎时按 [01-解析引擎/解析引擎详细功能与架构设计.md](01-解析引擎/解析引擎详细功能与架构设计.md) 的模块清单与接口落地。
5. **UI 与交互**：实现或验收 UI 时参考 [02-可视化界面/UI与交互逻辑.md](02-可视化界面/UI与交互逻辑.md)；按模块实现 UI 时按 [02-可视化界面/可视化界面详细功能与架构设计.md](02-可视化界面/可视化界面详细功能与架构设计.md) 的模块清单与 postMessage 协议落地。
6. **Agent 实现**：实现 Agent 服务时按 [03-Agent模块/Agent模块详细功能与架构设计.md](03-Agent模块/Agent模块详细功能与架构设计.md) 的模块清单与接口落地（每模块约 500-800 行）；协议与请求/响应以 [03-Agent模块/接口与协议.md](03-Agent模块/接口与协议.md) 为准。
