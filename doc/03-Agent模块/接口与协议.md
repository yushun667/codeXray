# AI Agent 模块：接口与协议

## 1. 服务形态

- **本地服务**：以 stdio、Unix socket 或 HTTP 形式对外提供；主仓库与侧边栏 AI 对话通过主仓库后端连接该服务。
- 推荐：Remote-SSH 下无端口转发时，**stdio** 或 **Unix socket**（工作区或临时目录下）更易用；若需跨机调试可选用 **HTTP**（localhost）。

## 2. 会话与记忆 ID 约定

- **session_id**：每个对话会话一个 ID，由主仓库或本模块生成，用于关联短期记忆与历史消息。
- **memory_id** / **chunk_id**：长期记忆或知识库片段的唯一标识，用于更新、删除与召回。
- 具体字段名与格式在实现时定（如 UUID 或时间戳+序列），并在本文档或附录中列出。

## 3. 请求/响应格式（概要）

以下为逻辑分类，具体 JSON schema 在实现时维护。

### 3.1 对话

- **请求**：`{ "action": "chat", "session_id": "...", "message": "用户输入", "context": { "file": "...", "symbol": "...", "selection": "..." } }`
- **响应**：流式时逐 chunk 返回文本或 SSE；非流式时 `{ "reply": "..." }`。可选附带 `memory_updated`、`cited_chunks` 等。

### 3.2 记忆

- **写入**：`{ "action": "memory_write", "session_id": "...", "content": "...", "tags": [...] }` → `{ "memory_id": "..." }`
- **召回**：`{ "action": "memory_recall", "session_id": "...", "query": "...", "limit": N }` → `{ "chunks": [...] }`

### 3.3 知识库

- **更新**（若由主仓库/解析结果驱动）：`{ "action": "kb_update", "source": "codexray", "items": [ { "id": "...", "text": "...", "meta": {} } ] }` → `{ "ok": true }`
- **检索**：多由 Agent 内部在生成回复时调用；若需单独接口：`{ "action": "kb_search", "query": "...", "limit": N }` → `{ "results": [...] }`

## 4. 错误与流式

- **错误**：响应中 `{ "error": "code", "message": "..." }` 或 HTTP 4xx/5xx + body。
- **流式**：建议用 SSE 或逐行 JSON（如 NDJSON），每行一个 chunk；主仓库将 chunk 顺序转交侧边栏 UI 展示。

## 5. 与主仓库的对接

- 主仓库在扩展激活时启动或连接 Agent 服务（根据配置：stdio 子进程、socket 或 HTTP 客户端）。
- 侧边栏用户发送消息 → 主仓库构造上述请求（含当前文件/符号上下文）→ 发送给 Agent 服务 → 将回复或流式 chunk 通过 VSCode 消息传给侧边栏 Webview。
- 可选「将当前回答或解析结果写入知识库」：主仓库发送 kb_update 或等价 action，或由 Agent 在对话中自行写入，依实现约定。
