# 知识库：树状知识结构与节点设计

本文档约定本地知识库的**树状摘要结构**、**树状检索**、**节点结构**，以及**多格式文档解析**（含 OCR）与**本地模型摘要**的要点。与 [模块功能说明](模块功能说明.md) §5、[接口与协议](接口与协议.md) §3.3、[Agent模块详细功能与架构设计](Agent模块详细功能与架构设计.md) 中 knowledge 模块配套使用。

---

## 1. 设计目标

- **树状摘要**：使用**本地部署模型**对资料库进行分层摘要，形成**树状知识结构**；上层节点为下层内容的摘要或目录，叶节点对应原文片段或段落。
- **树状检索**：支持沿树的**层级检索**（从根向下或按层级展开）、**路径检索**（从某节点向下/向上）、**语义检索**（与向量/关键词结合，命中节点后可按树展开上下文）。
- **多格式文档**：支持 **PDF、Word（.doc/.docx）、Markdown** 等常用格式；文档解析需支持**图片/扫描件中的文字**，通过 **OCR 等开源库** 实现。
- **节点结构**：每个树节点有唯一标识、摘要文本、层级、父/子引用、源文档与位置等，便于存储、检索与展示。

---

## 2. 树状知识结构

### 2.1 总体形态

- 每个**文档**或每个**资料库集合**对应一棵或多棵**摘要树**（可先按文档建树，再可选按集合合并）。
- 树根：整篇文档或集合的**顶层摘要**（由本地模型生成）。
- 中间层：按**章节/段落/块**逐层摘要，每层节点为该子树内容的摘要；层数可配置（如 3～5 层）。
- 叶节点：对应**原文片段**（如段落、列表项、表格摘要）；可保留原文文本与在源文件中的位置（页码、偏移、行号等），便于溯源与高亮。

```text
                    [根：文档摘要]
                   /    |    \    ...
            [章1摘要] [章2摘要] [章3摘要]
             /  \       |        |
        [节1.1][节1.2] [节2.1]  [节3.1]
          |      |       |         |
        [片段] [片段]  [片段]   [片段]
```

- **构建方式**：由**本地部署的摘要模型**（或配置的 LLM）对原文做**分层摘要**：先对全文生成根摘要，再对划分后的块（按标题/段落切分）逐层生成子节点摘要，直至叶节点挂载原文片段。划分与摘要策略在实现时定（如按标题层级、按固定长度块、或混合）。

### 2.2 树状检索结构

- **层级检索**：按层级（level）过滤节点，如「只查第 2 层」「只查叶节点」。
- **路径检索**：给定节点 id，返回从根到该节点的路径（breadcrumb），或该节点子树下所有节点；用于展开「树状搜索」结果。
- **语义/关键词检索**：在节点文本（摘要或原文）上做向量检索或关键词检索，返回命中节点列表；每个结果可附带**路径**与**子树摘要**，形成树状上下文供 RAG 使用。
- **混合**：先语义/关键词检索得到候选节点，再按树向上取父节点摘要或向下取子节点，组装成树状上下文返回。

## 3. 节点结构（Node Schema）

每个树节点在存储与协议中建议包含以下字段（实现时可按需增删，名称以英文为准便于实现）。

| 字段 | 类型 | 说明 |
|------|------|------|
| **id** | string | 节点唯一标识（如 UUID 或 `doc_id:path`，全局唯一） |
| **tree_id** | string | 所属树的 ID（如文档 id 或集合 id） |
| **parent_id** | string? | 父节点 id；根节点为 null 或空 |
| **children_ids** | string[] | 子节点 id 列表（可选冗余，便于遍历） |
| **level** | number | 层级深度，根为 0，依次递增 |
| **title** | string? | 节点标题（若来自标题层级或自动生成的小标题） |
| **summary** | string | 本节点摘要文本（本地模型生成）；叶节点可为原文片段或短摘要 |
| **source** | object? | 源文档与位置，见下表 |
| **span** | object? | 在原文中的范围（用于高亮/溯源），见下表 |
| **meta** | object? | 扩展元数据（格式、语言、创建时间等） |

**source** 建议字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| doc_id | string | 源文档唯一 id（如文件 path 的 hash 或入库 id） |
| format | string | 文档格式：`pdf`、`doc`、`docx`、`md` |
| path | string? | 文件路径或 URI（可选，便于打开原文） |
| page | number? | 页码（PDF）；或 null |
| offset_start | number? | 在原文中的起始偏移（字符或 token） |
| offset_end | number? | 结束偏移 |

**span**（可选）：用于前端高亮或精确溯源。

| 字段 | 类型 | 说明 |
|------|------|------|
| start_line | number? | 起始行（Markdown/文本） |
| end_line | number? | 结束行 |
| start_offset | number? | 行内或全局起始偏移 |
| end_offset | number? | 结束偏移 |

- **叶节点**：若直接挂载原文，`summary` 可存原文片段，`source`/`span` 必填以便溯源；中间节点以 `summary` 为主，`source` 可存「代表范围」（如整章对应的页码区间）。

## 4. 文档格式与解析

### 4.1 支持的格式

- **PDF**：正文提取 + **图片/扫描页的 OCR**；需区分「文本型 PDF」与「扫描型 PDF」，后者依赖 OCR。
- **Word**：**.doc**（可选，需兼容库）、**.docx**（推荐）；提取段落、标题、表格（可转为文本或表格摘要）。
- **Markdown**：直接解析；按标题层级与段落划分，便于与树层级对应。

### 4.2 解析与 OCR 实现要点

- **文档解析**应使用**开源库**实现，不绑定商业组件；Rust 生态可考虑：
  - **PDF**：`pdf_extract`、`poppler` 绑定、或 `pdf-rs` 等提取文本；**OCR** 可调用 **Tesseract**（通过 `tesseract` crate 或子进程）、或 **ocrs** 等 Rust OCR 库，对图片页或嵌入图片做文字识别。
  - **Word**：`.docx` 为 ZIP+XML，可用 `docx-rs`、`calamine`（部分）或其它开源解析库提取段落与标题。
  - **Markdown**：`pulldown-cmark` 等解析为 AST，按标题层级与块划分。
- **流程**：入库时对每个文档（1）按格式选择解析器；（2）若为 PDF 且某页无文本层，则对该页渲染为图后调用 OCR；（3）得到「纯文本 + 块级结构（标题/段落/表）」；（4）按块划分后交给**树状摘要**模块，调用本地模型生成各层摘要并建树。
- **存储**：解析得到的原始块与树节点一并持久化；树节点表/集合需包含上述**节点结构**，并建索引（tree_id、parent_id、level、doc_id、以及 summary 的向量或全文索引）。

## 5. 本地模型与树状摘要流程

- **本地部署模型**：用于对「原文块」或「下层摘要」做**摘要生成**，不要求与对话 LLM 同一模型；可配置为专用摘要模型（如小参数量的 summarization 模型），以控制延迟与成本。
- **建树流程**（概要）：
  1. 文档解析得到**块序列**（带层级时按标题，否则按段落或固定长度）。
  2. **自底向上**或**自顶向下**：  
     - 自顶向下：先对全文生成根摘要，再对块分组生成子节点摘要，递归直到叶节点挂载原文。  
     - 自底向上：叶节点先存原文片段，再对相邻/同层节点两两或分组做摘要得到父节点，直至根。
  3. 每个节点写入**节点表**，字段符合 §3 节点结构；建立 **parent_id / children_ids** 与 **tree_id**、**level** 索引。
- **增量更新**：文档变更时可按 doc_id 删除旧树再重建，或仅对变更段落对应子树重算（实现时定）。

## 6. 与现有 doc 的对应

- **模块功能说明** §5：本地知识库能力扩展为「树状摘要 + 树状检索 + 多格式 + OCR」。
- **接口与协议** §3.3：kb_update 可接受「文件路径 + 格式」触发解析与建树，或接受已解析的 items；kb_search 响应可带 **tree_path**、**node** 结构；可选 **kb_tree_get**（按 tree_id 返回树结构或某节点子树）。
- **Agent 详细设计** knowledge 模块：增加 **tree/**（建树、节点 CRUD、层级/路径查询）、**doc_parser/**（PDF/Word/MD 解析与 OCR 调用）；index 存储节点与树；search 支持树状检索与语义检索结合。

---

## 7. 索引与检索约定（摘要）

- **节点表**：至少包含 id、tree_id、parent_id、level、summary、source、doc_id；对 summary 建全文或向量索引；对 tree_id、parent_id、level、doc_id 建索引以支持树遍历与过滤。
- **检索**：  
  - **kb_search(query, limit)**：在节点 summary（及可选原文）上做语义/关键词检索，返回 **KbResult** 列表，每条可带 **node**（含 id、summary、level、source）、**path**（从根到该节点的 id 或 title 列表）、**tree_id**。  
  - **kb_tree_get(tree_id, node_id?)**：返回整棵树或从某节点起的子树（节点列表或嵌套结构），供前端树状展示或上下文组装。
