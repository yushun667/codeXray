# AI Agent 模块：模块功能说明

## 1. 模块定位

- **子仓库**：AI Agent 管理模块（如 `codexray-agent`）。
- **技术**：Rust 实现。
- **职责**：Agent 记忆管理、多 Agent 编排、持久化更新记忆、本地知识库；对外提供本地服务供主仓库与侧边栏 AI 对话 UI 使用。

## 2. 功能列表

| 需求编号 | 功能 | 说明 |
|----------|------|------|
| 3.1 | 记忆管理 | Agent 的短期/长期记忆结构、读写 API；与 UI 对接时由主仓库转发请求，本模块负责存储与召回 |
| 3.1 | 多 Agent 编排 | 多 Agent 分工与调度策略（流水线、路由、协作等），在 doc 中约定模型；对外呈现为统一对话或任务接口 |
| 3.1 | 持久化更新记忆 | 记忆落盘、增量更新、版本或快照策略（如按会话或按时间） |
| 3.1 | 本地知识库 | 与 CodeXray 解析结果/文档的索引与检索（如 RAG），供 Agent 生成回复时使用；可对接解析引擎 DB 或主仓库提供的摘要/片段 |

## 3. 记忆模型（约定要点）

- **短期记忆**：当前会话内的对话轮次、当前上下文（如当前文件/符号），可放在内存或短期存储。
- **长期记忆**：跨会话的要点、用户偏好、重要结论等；持久化到本地存储（如 SQLite 或嵌入式 KV/向量库）。
- **读写 API**：主仓库或协议层调用「写入记忆」「按会话/标签召回记忆」等接口；实现时在《接口与协议》中定义请求/响应格式。

## 4. 编排模型（约定要点）

- 多 Agent 编排策略需在实现时在 doc 中明确，例如：
  - **流水线**：请求依次经过多个 Agent（如理解 → 检索知识库 → 生成）。
  - **路由**：按意图或类型将请求分发给不同 Agent。
  - **协作**：多 Agent 并行或迭代完成子任务再汇总。
- 对外可暴露为单一「对话」或「任务」入口，内部由本模块负责编排与状态管理。

## 5. 本地知识库

- 数据来源：CodeXray 解析结果（如符号、调用关系、类关系摘要）、用户文档或注释等。
- 能力：索引（如向量化或关键词）、检索（RAG 检索相关片段）、可选地由主仓库在解析完成后推送摘要或由本模块定期拉取。
- 存储：与记忆分离或共用后端（如 SQLite + 扩展、或嵌入式向量库），在《接口与协议》中约定更新与查询接口。

## 6. 与主仓库的边界

- 主仓库负责：建立与 Agent 服务的连接（stdio 或 socket/HTTP）、将侧边栏用户输入与上下文（当前文件/符号）转发给本模块、将回复（含流式）转交给侧边栏 UI、可选地触发「写入知识库」等。
- 本模块负责：记忆的读写与持久化、多 Agent 编排、知识库检索与更新、生成回复；不负责 VSCode UI 渲染。
- **Remote-SSH**：本模块以二进制或源码形式在远程主机运行，主仓库在远程扩展主机上连接本地（远程机上的）Agent 服务，行为与本地一致。
